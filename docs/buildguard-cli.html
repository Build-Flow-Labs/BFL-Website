BuildGuard CLI

Compliance scanning, auto-remediation, and evidence logging for GitHub organizations.

Overview

BuildGuard is a Go-based CLI that scans GitHub organizations for compliance violations across 10 built-in policies. It evaluates repository configurations, branch protection rules, file contents, and CI/CD settings against security and governance standards.

Results are mapped to 6 compliance frameworks (SOC2, SOX, NIST 800-53, ISO 27001, PCI-DSS, FedRAMP) covering 33 specific controls. Findings are recorded in a PostgreSQL evidence ledger, and violations can be auto-remediated via pull requests.

For the full product overview including policy details and framework mappings, see the BuildGuard product page.

Installation

terminal
# Build from source
git clone https://github.com/Build-Flow-Labs/Build-Guard.git
cd Build-Guard
go build -o buildguard ./cmd/buildguard
# Verify installation
./buildguard version
buildguard version 1.0.0

Commands

buildguard scan

Runs a compliance audit across all repositories in a GitHub organization. Evaluates 10 policies, records results in the evidence ledger, and optionally opens auto-remediation PRs.

# Basic scan
buildguard scan --org your-org
# Dry run (no DB, no PRs)
buildguard scan --org your-org --dry-run --skip-db
# Export reports
buildguard scan --org your-org --export-html report.html --export-json report.json
# With config file
buildguard scan --org your-org --config buildguard.json
Flags:
  • • --org, -o — GitHub organization to scan (required)
  • • --config, -c — Path to buildguard.json configuration file
  • • --dry-run — Run without making changes or creating remediation PRs
  • • --skip-db — Skip recording results to the database
  • • --export-json <path> — Export results as a JSON report
  • • --export-html <path> — Export results as an HTML report

buildguard init

Interactive setup wizard that creates a buildguard.json configuration file. Prompts for organization name, default branch, reviewer, auto-remediation settings, and per-policy enablement.

$ buildguard init
BuildGuard Configuration Wizard
GitHub Organization: acme-corp
Default branch [main]: main
Default reviewer: @platform-team
Enable auto-remediation? [Y/n]: Y
Enable POL-SEC-01 (Docker Root User)? [Y/n]: Y
Enable POL-GOV-02 (CODEOWNERS)? [Y/n]: Y
...
Configuration saved to buildguard.json

buildguard validate

Validates a buildguard.json configuration file and displays the parsed settings.

$ buildguard validate --config buildguard.json
Configuration is valid.
Organization: acme-corp
Policies enabled: 10/10
Auto-remediation: enabled
Flags:
  • • --config — Path to the configuration file to validate

buildguard version

Displays the current BuildGuard version.

$ buildguard version
buildguard version 1.0.0

Configuration

BuildGuard reads from a buildguard.json file. Generate one with buildguard init or create it manually:

buildguard.json
{
"organization": {
"name": "your-org",
"default_branch": "main"
},
"policies": {
"docker_root_user": { "enabled": true, "severity": "critical" },
"code_owners": { "enabled": true, "severity": "warning" },
"secret_exposure": { "enabled": true, "severity": "critical" },
"branch_protection": { "enabled": true, "severity": "critical" },
"sox_separation": { "enabled": true, "severity": "critical" },
"ci_pipeline": { "enabled": true, "severity": "warning" },
"license_file": { "enabled": true, "severity": "warning" },
"security_policy": { "enabled": true, "severity": "warning" },
"actions_pinning": { "enabled": true, "severity": "critical" },
"default_branch_naming": { "enabled": true, "severity": "info" }
},
"remediation": {
"enabled": true,
"branch_prefix": "buildguard/fix-",
"default_reviewer": "@your-team"
},
"filter": {
"include_repos": [],
"exclude_repos": [],
"exclude_archived": true
},
"custom_policies": {
"enabled": false,
"policies_dir": "./policies/custom"
}
}

Environment Variables

GITHUB_TOKEN GitHub access token (required for scan)
DATABASE_URL PostgreSQL connection string for evidence ledger
BUILDGUARD_ORG Default organization name
BUILDGUARD_DEFAULT_BRANCH Default branch to check
BUILDGUARD_SLACK_WEBHOOK Slack webhook URL for notifications
BUILDGUARD_CUSTOM_POLICIES_DIR Directory for custom .rego policy files

GitHub Action

Use Build-Flow-Labs/Build-Guard@v1 in your workflows for automated compliance scanning on every push or PR.

.github/workflows/compliance.yml
name: BuildGuard Compliance
on: [push, pull_request]
jobs:
compliance:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v4
- uses: Build-Flow-Labs/Build-Guard@v1
with:
github_token: ${{ secrets.GITHUB_TOKEN }}
organization: your-org
config_path: .github/buildguard.json
fail_on_violation: true
database_url: ${{ secrets.DATABASE_URL }}

Action Inputs

Input Required Description
github_tokenYesGitHub token with repo access
organizationYesGitHub organization to scan
config_pathNoPath to buildguard.json config file
policiesNoComma-separated policy list (empty = all)
fail_on_violationNoFail the workflow if violations are found
dry_runNoRun without creating remediation PRs
database_urlNoPostgreSQL connection string for audit logging

Action Outputs

repos_scanned Number of repositories scanned
violations_found Number of violations found
scan_result JSON summary of scan results
