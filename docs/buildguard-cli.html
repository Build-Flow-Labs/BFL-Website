BuildGuard CLI

v1.0.0 — The Sovereign Auditor

Overview

BuildGuard is a standalone, Go-based CLI designed for high-assurance environments. It provides a unified interface for software supply chain integrity—validating build environments, enforcing policies, generating PBOMs, and signing artifacts.

Unlike traditional security scanners that operate after the fact, BuildGuard integrates at the Pre-flight, Runtime, and Attestation phases of your CI/CD pipeline. It doesn't just scan; it attests, signs, and enforces.

Installation

terminal
# Install via Build Flow Labs registry
curl -sSfL https://get.buildflowlabs.io/install.sh | sh
# Verify installation
buildguard --version
buildguard v1.0.0 (go1.22, linux/amd64)

Commands

buildguard audit

Scans your repository for CI configuration vulnerabilities before you push. Analyzes workflow YAML files for security anti-patterns.

# Usage
buildguard audit --path ./.github/workflows
Checks performed:
  • • OIDC token usage validation
  • • Unpinned action detection
  • • Over-privileged token identification
  • • Shell-injection risk analysis
  • • Secret exposure scanning

buildguard sign

Generates the PBOM and signs the build artifact using Sigstore/Cosign. Produces a cryptographically signed JSON file containing the chain of custody.

# Usage
buildguard sign --artifact ./dist/app --out ./dist/pbom.json
Flags:
  • • --artifact — Path to the build artifact to sign
  • • --out — Output path for the PBOM attestation file
  • • --key — Optional: path to a custom signing key

buildguard policy check

Validates your environment against internal Build Flow Labs standards or custom Rego policy files.

# Usage
buildguard policy check --rego ./policies/nist-800-161.rego
Flags:
  • • --rego — Path to custom Rego policy file
  • • --strict — Fail on warnings (default: fail on errors only)
  • • --output — Output format: json, table, or summary

buildguard pbom

Generates a Pipeline Bill of Materials by scanning the current CI environment, capturing runner metadata, toolchain versions, and environment context.

# Generate a standalone PBOM
buildguard pbom --generate --path ./pipelines
# Generate and attach to an artifact
buildguard pbom --generate --attach ./dist/app
Captured metadata:
  • • Git SHA and branch
  • • Runner UUID and environment type
  • • OIDC subject identity
  • • Toolchain versions (Go, Node, Python, etc.)
  • • Active policy rules and results
  • • Timestamp and build duration

buildguard check

Full enterprise-mode audit combining policy evaluation, PBOM generation, and attestation signing in a single execution.

# Full enterprise audit
buildguard check --mode enterprise --pbom-enabled
# Target a specific repository
buildguard check --target github.com/org/repo

Configuration

BuildGuard reads from a .buildguard.yaml file in your repository root:

.buildguard.yaml
# BuildGuard Configuration
version: "1.0"
mode: enterprise
policies:
path: ./policies/
strict: true
pbom:
enabled: true
output: ./dist/pbom.json
sign: true
sbom:
format: cyclonedx
output: ./dist/sbom.json
runners:
require_ephemeral: true
require_oidc: true

Examples

GitHub Actions Integration

.github/workflows/buildguard.yml
name: BuildGuard Audit
on: [push, pull_request]
jobs:
audit:
runs-on: ubuntu-latest
permissions:
id-token: write
contents: read
steps:
- uses: actions/checkout@v4
- name: Install BuildGuard
run: curl -sSfL https://get.buildflowlabs.io/install.sh | sh
- name: Run Audit
run: buildguard check --mode enterprise --pbom-enabled
